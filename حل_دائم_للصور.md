# حل دائم لمشكلة الصور المفقودة

## المشكلة
عند إضافة صور جديدة في Docker image، لا تظهر على السيرفر لأن vollume mapping يخفي محتويات الـ imagee.

## الحل المطبق

تم تعديل `docker-compose.dev.yml` لإزالة volume mapping للـ `public` folder من app container. الآن الصور موجودة في الـ image ويمكن نسخها تلقائياً.
و
## خطوات التطبيق على السيرفر

### 1. تحديث docker-compose.dev.yml

انسخ الملف المحدث إلى السيرفر:
```bash
cd /var/www/nemo-tours-dev
git pull origin dev
# أو انسخ الملف يدوياً
```

### 2. إيقاف الـ containers

```bash
cd /var/www/nemo-tours-dev
docker-compose -f docker-compose.dev.yml down
```

### 3. نسخ السكريبت

انسخ `sync-public-files.sh` إلى السيرفر وجعله قابل للتنفيذ:
```bash
chmod +x sync-public-files.sh
```

### 4. سحب آخر Docker image

```bash
docker-compose -f docker-compose.dev.yml pull app
```

### 5. بدء الـ containers

```bash
docker-compose -f docker-compose.dev.yml up -d
```

### 6. نسخ الصور من الـ image

```bash
./sync-public-files.sh
```

أو يدوياً:
```bash
docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/. ./public/landing-page/assets/
chown -R www-data:www-data ./public
chmod -R 755 ./public
```

## الحل التلقائي (عند كل deployment)

أضف هذا الأمر في CI/CD pipeline أو deployment script:

```bash
# بعد docker-compose up -d
docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/. /var/www/nemo-tours-dev/public/landing-page/assets/
chown -R www-data:www-data /var/www/nemo-tours-dev/public
```

## الحل الأفضل: إضافة إلى CI/CD

أضف هذه الخطوة في `.github/workflows/dev.yaml` أو deployment script:

```yaml
- name: Sync public files from image
  run: |
    docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/. /var/www/nemo-tours-dev/public/landing-page/assets/
    chown -R www-data:www-data /var/www/nemo-tours-dev/public
```

## ملاحظات مهمة

1. **الصور الجديدة**: بعد كل deployment جديد أو عند إضافة صور جديدة، قم بتشغيل `sync-public-files.sh`
2. **الصلاحيات**: تأكد من أن الصلاحيات صحيحة (`www-data:www-data` أو `755`)
3. **التحقق**: اختبر الصور بعد النسخ:
   ```bash
   curl -I https://nemotours.duckdns.org/landing-page/assets/img/hero_bg.jpg
   ```

## حل بديل: استخدام Git للصور

إذا كنت تريد حل أكثر استقراراً، أضف جميع الصور إلى Git repository:

```bash
git add public/landing-page/assets/img/
git commit -m "Add landing page images"
git push origin dev
```

ثم على السيرفر:
```bash
git pull origin dev
```

هذا يضمن أن الصور موجودة دائماً على السيرفر.

# حل مشكلة الصور المفقودة على السيرفر

## المشكلة
الصور موجودة في Docker image وتعمل محلياً، لكنها لا تظهر على السيرفر (`404 Not Found`) لأنها غير موجودة في المجلد المحلي على السيرفر.

**السبب:** 
- الـ volume mapping في `docker-compose.dev.yml` يربط المجلد المحلي بالـ container
- nginx يقرأ الملفات من المجلد المحلي على السيرفر (`/var/www/nemo-tours-dev/public/`)
- الصور موجودة في Git لكن لم يتم نسخها بشكل صحيح على السيرفر

## الحل السريع (على السيرفر)

قم بتشغيل هذه الأوامر على السيرفر:

```bash
cd /var/www/nemo-tours-dev

# 1. سحب آخر التحديثات من Git
git pull origin dev

# 2. التأكد من وجود الصور
ls -la public/landing-page/assets/img/hero_bg.jpg

# 3. إذا لم تكن موجودة، انسخها من الـ container
docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/img/. /var/www/nemo-tours-dev/public/landing-page/assets/img/

# 4. تعديل الصلاحيات
chown -R www-data:www-data /var/www/nemo-tours-dev/public/landing-page/assets/img
chmod -R 755 /var/www/nemo-tours-dev/public/landing-page/assets/img

# 5. إعادة تشغيل nginx
docker restart nemotours_nginx_dev
```

## استخدام السكريبت التلقائي

يمكنك استخدام السكريبت الجاهز:

```bash
cd /var/www/nemo-tours-dev
bash fix-missing-images-server.sh
```

أو انسخ محتوى `fix-missing-images-server.sh` إلى السيرفر وشغله.

## التحقق من الحل

بعد تطبيق الحل، اختبر الصورة:

```bash
curl -I https://nemotours.duckdns.org/landing-page/assets/img/hero_bg.jpg
```

يجب أن يعطي:
```
HTTP/1.1 200 OK
```

بدلاً من:
```
HTTP/1.1 404 Not Found
```

## الحل الدائم

لتجنب هذه المشكلة في المستقبل:

1. **تأكد من أن جميع الصور موجودة في Git:**
   ```bash
   git add public/landing-page/assets/img/
   git commit -m "Add missing images"
   git push origin dev
   ```

2. **على السيرفر، تأكد من pull التحديثات بعد كل deployment:**
   ```bash
   cd /var/www/nemo-tours-dev
   git pull origin dev
   ```

3. **أضف خطوة في CI/CD لنسخ الصور تلقائياً** (إذا كانت موجودة في Docker image فقط)

## ملاحظات

- الصور موجودة في Git repository ✅
- المشكلة في السيرفر فقط - لم يتم نسخ الملفات بشكل صحيح
- الحل السريع: نسخ من container أو pull من Git
- الحل الدائم: التأكد من أن deployment process ينسخ جميع الملفات

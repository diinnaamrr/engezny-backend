## كيفية العمل

### تلقائياً (عند كل deployment)
عندما تدفع كود جديد إلى branch `dev`:
1. يتم بناء Docker image جديد
2. يتم deployment على السيرفر
3. **تلقائياً** يتم نسخ الصور من image إلى host
4. الصور تظهر مباشرة! 🎉

### يدوياً (عند الحاجة)
إذا أردت نسخ الصور يدوياً:
```bash
cd /var/www/nemo-tours-dev
./sync-public-files.sh
```

أو:
```bash
docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/. ./public/landing-page/assets/
chown -R www-data:www-data ./public
```

## الخطوات التالية على السيرفر

### 1. تحديث الملفات
```bash
cd /var/www/nemo-tours-dev
git pull origin dev
```

### 2. إعادة تشغيل Containers
```bash
docker-compose -f docker-compose.dev.yml down
docker-compose -f docker-compose.dev.yml pull
docker-compose -f docker-compose.dev.yml up -d
```

### 3. نسخ الصور (للمرة الأولى)
```bash
# انتظر حتى يكون container جاهز
sleep 5

# انسخ الصور
docker cp nemotours_app_dev:/var/www/html/public/landing-page/assets/. ./public/landing-page/assets/
chown -R www-data:www-data ./public
chmod -R 755 ./public
```

### 4. التحقق
```bash
curl -I https://nemotours.duckdns.org/landing-page/assets/img/hero_bg.jpg
```

يجب أن يعطي `200 OK` ✅

## ملاحظات مهمة

1. **الصور الجديدة**: بعد كل deployment جديد، الصور ستُنسخ تلقائياً من workflow
2. **إذا لم تظهر الصور**: قم بتشغيل `sync-public-files.sh` يدوياً
3. **الصلاحيات**: تأكد من أن الصلاحيات صحيحة (`www-data:www-data` أو `755`)

## الحل البديل (إذا لم يعمل التلقائي)

إذا لم تعمل النسخ التلقائي، يمكنك:
1. إضافة الصور إلى Git repository
2. على السيرفر: `git pull origin dev`
3. الصور ستكون موجودة مباشرة من Git
